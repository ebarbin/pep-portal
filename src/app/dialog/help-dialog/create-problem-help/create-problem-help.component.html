Crear/Editar Ejercicio

Desde aquí podrás crear/editar los ejercicios. A continuación una descripción de los campos:

Título del ejercicio

Utilizado para identificar el ejercicio. Requerido.


Primitivas

Permite buscar y seleccionar primitivas. Ten en cuenta que la busqueda se realiza por el nombre de la primitiva.


Enunciado

Descripción del ejercicio a realizar. Ten en cuenta de agregar toda la información necesaria para resolverlo sobre todo en los casos donde agreges datos o funciones en la pre ejecución. Requerido


Pre Ejecución

Aquí debes definir los datos y/o funciones que quieras dejar disponible a los alumnos y que son necesarias o no para la resolución del ejercicio.


Solución

Esta sección sirve para probar la validación del ejercicio. Es opcional.


Pos Ejecución

Aquí debes definir la función que valide el ejercicio. Una vez validado el ejercicio debes actualizar la propiedad state del objeto global __result:

  • true Validación automática. Ejercicio resuelto correctamente.
  • false Validación automática. Ejercicio resuelto incorrectamente.
  • null Validación manual. En este caso el ejercicio debe ser corregido por el docente desde la sección de correcciones.

El objeto __result cuenta, además, con la propiedad message que sirve para informar al alumno del resultado de su solución.


Aquí se muestra el contexto de aplicación completo. Este contexto esta divido en los siguientes bloques:

  • Pre Ejecución Estática Este bloque no se puede modificar, lo agrega el sistema y es siempre el mismo.
  • Pre Ejecución Este bloque lo define el docente. Y como vimos, sirve para dejar información o funciones necesarias en el ejercicio.
  • Primitivas Aqui se agregan las primitivas. Que no son más que funciones simples.
  • Solución Este bloque sirve para probar el ejercicio y lo debe definir el docente aunque no es requerido.
  • Pos Ejecución Este bloque lo define el docente. Y como vimos, sirve para determinar el resultado del ejercicio.
  • Pos Ejecución Estática Este bloque no se puede modificar, lo agrega el sistema y es siempre el mismo.

Realiza la prueba del ejercicio. Basicamente se prueba la pos ejecución o validación del ejercicio.
